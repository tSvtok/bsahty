critical bugs & issues to fix
CRITICAL
Zero test coverage — no PHPUnit tests written
All models, controllers, API endpoints are untested. PhpUnit is configured but no test file exists.
CRITICAL
Messaging is non-functional for real use
Conversation structure exists but real-time WebSocket delivery, read receipts, typing indicators, and unread tracking are all missing.
CRITICAL
No file upload system anywhere in the app
No image upload for posts, events, or user avatars. Backend storage config, validation and CDN integration absent.
HIGH
Token expiration not handled — users get silent 401s
No refresh token mechanism. Sanctum tokens expire and the frontend axios interceptor has no recovery logic.
HIGH
Map page spots panel not responsive on mobile
Leaflet map and spots side-panel overlap on small screens. No mobile-specific layout breakpoint defined.
HIGH
ProfilePage stats grid crowded / broken on mobile
Fixed-height cover image and multi-column stats grid do not scale. Images are not responsive.
HIGH
PostCard like action may silently fail (API mismatch)
Reaction endpoint expects polymorphic type field; frontend may be sending incorrect payload format.
HIGH
No email verification or password reset
Users can register with any email and there is no recovery path if they lose their password.
HIGH
Seed data nearly empty (25%)
Only a single sample spot seeded. No test users, events, questions, friendships, or coordinates for map testing.
MEDIUM
Inconsistent API response formats across controllers
Some endpoints return data under a root key, others do not. Pagination metadata not fully consumed on the frontend.
MEDIUM
Missing authorization checks in some controller methods
Ownership checks (e.g. can user X delete post Y?) not consistently applied. Any authenticated user may be able to mutate others' data.
MEDIUM
Sidebar doesn't adapt on tablet (768–1024px)
Fixed desktop sidebar renders incorrectly in the tablet breakpoint range, pushing content.
MEDIUM
Modal dialogs have no responsive padding on small screens
Modals (CreatePost, CreateEvent, EditProfile) overflow on viewports under ~400px wide.
MEDIUM
No rate limiting or caching on the API
All endpoints are unthrottled. Repeated identical requests hit the DB every time. Open to abuse.
MEDIUM
CORS configuration not reviewed for production
Default Laravel CORS may expose all origins. Needs explicit allowed-origin whitelist before deploy.
MEDIUM
UI color / badge contrast too low — accessibility fail
Badge colors are too light. Hover states are not uniform. No focus rings on interactive elements. Fails WCAG AA contrast ratios.
MEDIUM
Date formatting inconsistent across components
Some components use raw ISO strings, others format — resulting in mismatched display across the app.
MEDIUM
Edit / delete own post not implemented
Users can create posts but cannot edit or delete them. No soft deletes configured on the model.
MEDIUM
Spot "Add custom spot" UI missing
The backend nearby-spot endpoint is ready but there is no frontend interface to let users submit new spots.
MEDIUM
Event RSVP / join functionality missing
Events display participant count but users cannot actually join/leave. RSVP state not tracked in DB.
FIXED
Spot lat/lng not exposed in API — map markers missing
PostGIS coordinates schema replaced with JSON; Spot model appends coordinates correctly.
FIXED
Navbar missing mobile menu
Mobile hamburger menu added to Navbar component.
